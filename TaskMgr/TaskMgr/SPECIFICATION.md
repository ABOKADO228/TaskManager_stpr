# Спецификация TaskMgr

## 1. Назначение

TaskMgr - учебный веб-прототип для управления группами, расписанием, домашними заданиями, событиями недели, заметками и комментариями.

Сервер на Bottle отдает шаблоны и статические файлы. Данные прототипа сохраняются в браузере через `localStorage` и `sessionStorage`.

## 2. Технологии

- Backend: Python, Bottle.
- Шаблоны: `views/*.tpl`.
- Стили: обычный CSS в `static/content/css/`.
- Клиентская логика: `static/scripts/week-calendar.js`.
- Сборка стилей: не используется.

SCSS/Sass в проекте отсутствует.

## 5. Роли

- Гость: видит стартовую страницу и может перейти к входу.
- Авторизованный пользователь: может создать группу или войти по коду.
- Староста `leader`: создает и редактирует события группы.
- Ученик `member`: просматривает события и оставляет комментарии.

## 6. Функциональность

- Регистрация сохраняет пользователя в `localStorage.taskmgr_users`.
- Вход создает `localStorage.taskmgr_session`.
- Создание группы назначает создателю роль `leader`.
- Присоединение по коду назначает роль `member`.
- События недели хранятся внутри группы.
- Староста может добавлять и редактировать события.
- Ученик может комментировать события.
- Заметки сохраняются отдельно для каждого пользователя.
- Страница `/orders` выводит список заказов из `data/orders.json`.
- Форма `/orders` добавляет новый заказ после серверной проверки.
- При ошибках поля формы сохраняют введенные значения.
- После успешного добавления форма очищается через redirect.
- Страница `/active-users` выводит список активных пользователей из `data/active_users.json`.
- Форма `/active-users` добавляет нового пользователя после серверной проверки.

## 6.1 Оформленные заказы

Объект заказа содержит:

- номер заказа;
- автора или клиента;
- описание;
- дату в формате `YYYY-MM-DD`;
- телефон в формате `+7XXXXXXXXXX`, `8XXXXXXXXXX` или с разделителями.

Проверки:

- номер обязателен, уникален и содержит 3-20 допустимых символов;
- автор обязателен и не короче 2 символов;
- описание обязательно и не короче 10 символов;
- дата обязательна, соответствует формату и не позже текущей даты;
- телефон обязателен и содержит корректный российский номер.

Заказы сортируются по дате от новых к старым.

## 6.2 Активные пользователи

Объект активного пользователя содержит:

- ник;
- описание активности;
- дату активности в формате `YYYY-MM-DD`;
- телефон в формате `+7XXXXXXXXXX`, `8XXXXXXXXXX` или с разделителями.

Проверки:

- ник обязателен, уникален и содержит 3-24 допустимых символа;
- описание обязательно и не короче 10 символов;
- дата активности обязательна, соответствует формату и не позже текущей даты;
- телефон обязателен и содержит корректный российский номер.

Пользователи сортируются по дате активности от новых к старым.

## 7. Хранение данных

| Ключ | Описание |
| --- | --- |
| `localStorage.taskmgr_users` | Аккаунты |
| `localStorage.taskmgr_session` | Текущая сессия |
| `localStorage.taskmgr_groups` | Группы, роли, события, комментарии |
| `localStorage.taskmgr_notes_<userId>` | Личные заметки |
| `localStorage.isAuth` | Флаг авторизации |
| `sessionStorage.tempAuth` | Временная сессия |
| `data/orders.json` | Файл заказов для страницы `/orders` |
| `data/active_users.json` | Файл активных пользователей для страницы `/active-users` |

## 8. Ограничения

- Нет серверной базы данных.
- Пароли хранятся в браузере в открытом виде.
- Проверка ролей выполняется на клиенте.
- Нет удаления событий.
- Нет переключения между несколькими группами.

## 9. Критерии приемки

- `/` загружает `preview.css` и все его импортируемые файлы без 404.
- `/reg-auth` загружает `auth-reg.css` и все его импортируемые файлы без 404.
- `/main` загружает `main.css` и все его импортируемые файлы без 404.
- `/orders` загружает `orders.css` и все его импортируемые файлы без 404.
- `/orders` добавляет валидный заказ в `data/orders.json`.
- `/orders` показывает ошибки валидации без очистки формы.
- `/active-users` загружает `active-users.css` и все его импортируемые файлы без 404.
- `/active-users` добавляет валидного пользователя в `data/active_users.json`.
- `/active-users` показывает ошибки валидации без очистки формы.
- Unit-тесты проверки даты и телефона выполняются через `python -m unittest discover -s tests`.
- В проекте нет папки `static/content/scss`.
- В `package.json` нет Sass-зависимости и SCSS-скриптов.
