# TaskMgr

TaskMgr - учебный веб-прототип для работы с учебными группами, недельным расписанием, событиями, комментариями, личными заметками, оформленными заказами и списком активных пользователей.

Проект построен на Python Bottle, HTML-шаблонах, обычном CSS и чистом JavaScript. Сервер отдает маршруты, шаблоны, статические файлы и хранит JSON-данные для страниц заказов и активных пользователей. Аккаунты, сессия, группы, роли, расписание, комментарии и заметки основного рабочего пространства пока хранятся в браузере через `localStorage` и `sessionStorage`.

## Возможности

- Стартовая страница `/`.
- Регистрация и вход на странице `/reg-auth`.
- Рабочее пространство `/main` после входа.
- Создание учебной группы с ролью старосты.
- Присоединение к группе по коду в роли ученика.
- Переключение между группами пользователя.
- Просмотр участников выбранной группы и их ролей.
- Ролевые права: староста создает, редактирует и удаляет события; ученик просматривает события и оставляет комментарии.
- Недельное расписание с типами событий: урок, домашнее, практика, контрольная, другое событие.
- Личные заметки пользователя.
- Комментарии к событиям группы.
- Страница оформленных заказов `/orders`.
- Страница активных пользователей `/active-users`.
- Серверная валидация форм заказов и активных пользователей.
- Ограничение дат текущим днем: будущие даты отклоняются серверной проверкой и ограничиваются в HTML-полях.
- Поиск по списку заказов и активных пользователей.
- Уведомление после успешного добавления записи.
- Кнопки очистки форм.
- Предварительный расчет рейтинга активности на странице активных пользователей.
- Unit-тесты для дат, телефонов, форм, владельцев заказов и рейтинга активности.

## Запуск

Установите Python-зависимости:

```powershell
pip install -r requirements.txt
```

Запустите сервер:

```powershell
python app.py
```

Откройте в браузере:

```text
http://localhost:5555
```

Хост и порт можно изменить через переменные окружения:

```powershell
$env:SERVER_HOST = "127.0.0.1"
$env:SERVER_PORT = "8080"
python app.py
```

Debug-режим можно включить аргументом `--debug` или переменной `SERVER_DEBUG`.

## Основные файлы

`app.py` - точка входа Bottle-приложения. Настраивает путь к шаблонам, подключает маршруты, отдает файлы из `static/` и запускает сервер.

`routes.py` - маршруты приложения:

- `/` - стартовая страница;
- `/reg-auth` - вход и регистрация;
- `/main` - рабочее пространство группы;
- `/orders` - оформленные заказы;
- `/active-users` - активные пользователи;
- `/static/<filepath:path>` - CSS, JavaScript, шрифты и другие статические файлы.

`order_service.py` - загрузка, сохранение, сортировка, фильтрация, валидация и сборка объектов заказов.

`active_user_service.py` - загрузка пользователей, расчет рейтинга активности, фильтрация активных пользователей, сохранение и валидация формы.

`static/scripts/week-calendar.js` - клиентская логика рабочего пространства: сессия, группы, роли, события недели, комментарии и заметки.

## Оформленные Заказы

Страница доступна по адресу:

```text
http://localhost:5555/orders
```

Перед просмотром нужно авторизоваться через `/reg-auth`. После входа клиентская страница сохраняет cookie `taskmgr_user_id`, `taskmgr_username` и `taskmgr_display_name`, чтобы сервер мог показать только личные заказы пользователя.

На странице `/orders`:

- GET загружает заказы из `data/orders.json`;
- неавторизованный пользователь перенаправляется на `/reg-auth?next=/orders`;
- отображаются только записи текущего пользователя по `owner_id`;
- список сортируется по дате от новых к старым;
- новые заказы добавляются через POST-форму;
- заказ сохраняется с `owner_id` и `owner_name`;
- форма проверяется на сервере;
- при ошибках введенные значения остаются в форме;
- после успешного добавления выполняется redirect на `/orders?saved=1`;
- параметр `saved=1` включает уведомление об успешном сохранении;
- поле даты ограничено текущим днем;
- список можно фильтровать через поле поиска;
- форму можно очистить кнопкой `Очистить`.

Обязательные поля заказа:

- номер;
- автор или клиент;
- описание;
- дата;
- телефон.

## Активные Пользователи

Страница доступна по адресу:

```text
http://localhost:5555/active-users
```

На странице `/active-users`:

- GET загружает пользователей из `data/active_users.json`;
- активные пользователи выбираются автоматически по рейтингу;
- новые пользователи добавляются через POST-форму;
- форма проверяется на сервере;
- при ошибках введенные значения остаются в форме;
- после успешного добавления выполняется redirect на `/active-users?saved=1`;
- параметр `saved=1` включает уведомление об успешном сохранении;
- дата активности ограничена текущим днем;
- список можно фильтровать через поле поиска;
- форма показывает предварительный расчет рейтинга;
- форму можно очистить кнопкой `Очистить`.

Обязательные поля пользователя:

- ник;
- описание;
- дата активности;
- телефон;
- количество созданных событий;
- количество комментариев;
- количество заметок;
- количество групп.

Формула рейтинга:

```text
события * 4 + комментарии * 2 + заметки + группы * 3 + бонус корректной даты
```

Пользователь считается активным, если рейтинг не ниже `10`.

## Данные В Браузере

- `localStorage.taskmgr_users` - аккаунты;
- `localStorage.taskmgr_session` - текущая сессия;
- `localStorage.taskmgr_groups` - группы, роли, события и комментарии;
- `localStorage.taskmgr_notes_<userId>` - личные заметки;
- `localStorage.isAuth` - флаг авторизации;
- `localStorage.username` - логин текущего пользователя;
- `localStorage.userRole` - вспомогательное поле роли;
- `localStorage.rememberMe` - признак сохранения входа;
- `sessionStorage.tempAuth` - временная сессия.

Cookie:

- `taskmgr_user_id` - id пользователя для серверной фильтрации заказов;
- `taskmgr_username` - логин пользователя;
- `taskmgr_display_name` - отображаемое имя пользователя.

## Стили

Проект использует обычный CSS без SCSS/Sass. Основные точки входа:

- `static/content/css/auth-reg/preview.css` - стартовая страница;
- `static/content/css/auth-reg/auth-reg.css` - вход и регистрация;
- `static/content/css/main/main.css` - главное рабочее пространство;
- `static/content/css/orders/orders.css` - страница заказов;
- `static/content/css/active-users/active-users.css` - страница активных пользователей.

CSS для заказов и активных пользователей разбит на маленькие файлы: `base.css`, `header.css`, `layout.css`, `form.css`, `list.css`, `responsive.css`.

## Тесты

Запуск unit-тестов:

```powershell
python -m unittest discover -s tests
```

Альтернативно, если установлен pytest:

```powershell
python -m pytest tests -p no:cacheprovider -q
```

Сейчас выполняется 28 тестов. Они проверяют:

- корректную дату относительно текущего дня;
- отклонение будущих дат;
- некорректный формат даты;
- корректный и некорректный российский телефон;
- валидацию формы заказа;
- валидацию формы активного пользователя;
- дубликаты номера заказа и ника пользователя;
- обязательные поля форм;
- сборку заказа с владельцем и без владельца;
- расчет рейтинга активности;
- сборку активного пользователя из формы;
- фильтрацию заказов по текущему пользователю.

Покрытие после последней проверки:

```text
active_user_service.py  73%
order_service.py        78%
TOTAL                   75%
```

Оставшиеся непокрытые участки в основном относятся к JSON load/save и helper-функциям путей.

## Ограничения

- Серверная база данных не подключена.
- Аккаунты, группы и расписание хранятся в браузере.
- Пароли в прототипе хранятся в открытом виде.
- Cookie авторизации не подписаны серверным секретом.
- Роли в рабочем пространстве проверяются на клиенте.
- Удаление группы пока не реализовано.
