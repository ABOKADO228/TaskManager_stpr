# TaskMgr

TaskMgr - учебный веб-прототип для работы с расписанием учебной группы, событиями, заметками, комментариями, оформленными заказами и активными пользователями. Проект построен на Python Bottle, HTML-шаблонах, обычном CSS и чистом JavaScript.

Сервер отвечает за маршруты, шаблоны, статические файлы, JSON-данные заказов и JSON-данные активных пользователей. Основная логика аккаунтов, групп, ролей и расписания в прототипе хранится в браузере через `localStorage`.

## Возможности

- Стартовая страница `/`.
- Регистрация и вход на странице `/reg-auth`.
- Сохранение созданных аккаунтов в браузере.
- Создание учебной группы с ролью старосты.
- Присоединение к группе по коду в роли ученика.
- Переключение между группами пользователя.
- Просмотр участников выбранной группы и их ролей.
- Ролевые права: староста редактирует и удаляет события, ученик просматривает события и комментирует.
- Недельное расписание с занятиями, домашними заданиями, практическими и другими событиями.
- Личные заметки пользователя.
- Комментарии к событиям.
- Страница оформленных заказов `/orders`.
- Страница активных пользователей `/active-users`.
- Серверная валидация форм заказов и активных пользователей.
- Поиск по списку заказов и списку активных пользователей.
- Уведомление после успешного добавления записи.
- Ограничение дат сегодняшним днем в формах.
- Кнопка очистки форм.
- Предварительный расчет рейтинга на странице активных пользователей.

## Запуск

Установите Python-зависимости:

```powershell
pip install -r requirements.txt
```

Запустите сервер:

```powershell
python app.py
```

Откройте в браузере:

```text
http://localhost:5555
```

Хост и порт можно изменить через переменные окружения:

```powershell
$env:SERVER_HOST = "127.0.0.1"
$env:SERVER_PORT = "8080"
python app.py
```

## Основные файлы

`app.py` - точка входа Bottle-приложения. Настраивает путь к шаблонам, подключает маршруты, отдает файлы из `static/` и запускает сервер.

`routes.py` - маршруты приложения:

- `/` - стартовая страница;
- `/reg-auth` - вход и регистрация;
- `/main` - основное рабочее пространство;
- `/orders` - оформленные заказы;
- `/active-users` - активные пользователи;
- `/static/<filepath:path>` - CSS, JavaScript, шрифты и другие статические файлы.

`order_service.py` - загрузка, сохранение, сортировка, фильтрация и валидация заказов.

`active_user_service.py` - загрузка пользователей, расчет рейтинга активности, фильтрация активных пользователей, сохранение и валидация формы.

`views/orders.tpl` - страница заказов с формой добавления, поиском, уведомлением об успешном сохранении и списком карточек.

`views/active-users.tpl` - страница активных пользователей с формой добавления, поиском, предпросмотром рейтинга и списком карточек.

`data/orders.json` - JSON-файл заказов.

`data/active_users.json` - JSON-файл пользователей и их метрик активности.

## Оформленные Заказы

Страница доступна по адресу:

```text
http://localhost:5555/orders
```

Перед просмотром нужно авторизоваться через `/reg-auth`. При успешном входе клиентская страница сохраняет cookie `taskmgr_user_id`, `taskmgr_username` и `taskmgr_display_name`, чтобы сервер мог показать только личные заказы пользователя.

На странице `/orders`:

- список заказов загружается Python-кодом из `data/orders.json`;
- отображаются только записи текущего пользователя по `owner_id`;
- новые заказы добавляются через POST-форму;
- заказ сохраняется с `owner_id` и `owner_name`;
- форма проверяется на сервере;
- при ошибках введенные значения остаются в форме;
- после успешного добавления выполняется redirect на `/orders?saved=1`;
- пользователь видит сообщение об успешном сохранении;
- дата в форме ограничена сегодняшним днем;
- список можно фильтровать через поле поиска;
- форму можно очистить кнопкой `Очистить`;
- заказы сортируются по дате от новых к старым.

Обязательные поля заказа:

- номер;
- автор или клиент;
- описание;
- дата;
- телефон.

## Активные Пользователи

Страница доступна по адресу:

```text
http://localhost:5555/active-users
```

На странице `/active-users`:

- общий список пользователей загружается Python-кодом из `data/active_users.json`;
- активные пользователи выбираются автоматически по рейтингу;
- новые пользователи добавляются через POST-форму;
- форма проверяется на сервере;
- при ошибках введенные значения остаются в форме;
- после успешного добавления выполняется redirect на `/active-users?saved=1`;
- пользователь видит сообщение об успешном сохранении;
- дата активности ограничена сегодняшним днем;
- список можно фильтровать через поле поиска;
- форму можно очистить кнопкой `Очистить`;
- рейтинг активности предварительно пересчитывается на странице до отправки формы.

Обязательные поля пользователя:

- ник;
- описание;
- дата активности;
- телефон;
- количество созданных событий;
- количество комментариев;
- количество заметок;
- количество групп.

Формула рейтинга:

```text
события * 4 + комментарии * 2 + заметки + группы * 3 + бонус корректной даты
```

Пользователь считается активным, если рейтинг не ниже `10`.

## Данные В Браузере

- `localStorage.taskmgr_users` - аккаунты;
- `localStorage.taskmgr_session` - текущая сессия;
- `localStorage.taskmgr_groups` - группы, роли, события и комментарии;
- `localStorage.taskmgr_notes_<userId>` - личные заметки;
- `localStorage.isAuth` - флаг авторизации;
- `localStorage.username` - логин текущего пользователя;
- `localStorage.userRole` - вспомогательное поле роли;
- `localStorage.rememberMe` - признак сохранения входа;
- `sessionStorage.tempAuth` - временная сессия.

Cookie:

- `taskmgr_user_id` - id пользователя для серверной фильтрации заказов;
- `taskmgr_username` - логин пользователя;
- `taskmgr_display_name` - отображаемое имя пользователя.

## Стили

Проект использует обычный CSS без SCSS/Sass. Основные точки входа:

- `static/content/css/auth-reg/preview.css` - стартовая страница;
- `static/content/css/auth-reg/auth-reg.css` - вход и регистрация;
- `static/content/css/main/main.css` - главное рабочее пространство;
- `static/content/css/orders/orders.css` - страница заказов;
- `static/content/css/active-users/active-users.css` - страница активных пользователей.

CSS для заказов и активных пользователей разбит на маленькие файлы: `base.css`, `header.css`, `layout.css`, `form.css`, `list.css`, `responsive.css`.

Тесты проверяют:

- корректность даты заказа;
- корректность даты активности;
- формат телефона;
- расчет рейтинга активности;
- фильтрацию заказов по текущему пользователю;
- сохранение владельца нового заказа.

## Ограничения

- Серверная база данных не подключена.
- Аккаунты, группы и расписание хранятся в браузере.
- Пароли в прототипе хранятся в открытом виде.
- Cookie авторизации не подписаны серверным секретом.
- Роли проверяются на клиенте.
- Удаление группы пока не реализовано.
